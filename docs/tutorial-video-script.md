# ATTRACT チュートリアル動画 台本（ストーリーボード）

> **目的①** 操作フローを再現し、ユーザビリティをシビアにチェック＆改善
> **目的②** 60代の経営者にも、使い方と価値がわかるように見える化
> **想定尺** 約8〜10分（シーン別に分割収録し、後で編集を推奨）
> **URL** https://attract-prototype.vercel.app/

## シーン 1：ダッシュボード（60秒）

### 操作手順
1. `https://attract-prototype.vercel.app/` を開く
2. ダッシュボード全体を見せる（KPIカード4つ）
3. 「要対応アクション」セクションをゆっくりスクロール
4. アクション項目の1つをクリック → 候補者詳細へ遷移

### ナレーション（案）
> ログインすると、まずダッシュボードが表示されます。
> 上部には「選考中の候補者数」「要対応アクション数」「Attract設計済み」
> 「フィードバックレター送付済み」の4つのKPIが並びます。
>
> 特に重要なのが「要対応アクション」。
> AIが選考状況を分析し、**今日やるべきこと**を優先順位つきで提示してくれます。
> 採用担当者は、このリストに沿って動くだけで漏れなく対応できます。

### ユーザビリティチェック項目
- [ ] KPI数値の意味が初見で理解できるか？ラベルは直感的か？
- [ ] 要対応アクションの優先度がパッと見でわかるか？（色分け・アイコン）
- [ ] アクション項目をクリックした時、期待通りのページに遷移するか？
- [ ] 「新卒1名、中途3名」の表示は、ターゲットユーザーに伝わるか？
- [ ] 日付・曜日表示は自然か？

## シーン 2：候補者を登録する（60秒）

### 操作手順
1. サイドバー「候補者管理」をクリック → `/candidates`
2. 候補者一覧を見せる（検索窓、フィルター）
3. 「候補者を登録」ボタンをクリック → `/candidates/new`
4. 「中途」カードを選択 → フォーム入力（名前、会社、職種など）
5. 「新卒」カードに切り替え → 学歴フィールドが表示される様子を見せる
6. 同意チェック → 登録ボタンを押す → 成功バナー

### ナレーション（案）
> サイドバーから「候補者管理」を開くと、現在選考中の全候補者が一覧できます。
> 名前や会社名で検索もできます。
>
> 新しい候補者を登録してみましょう。
> まず「中途」か「新卒」を選択します。
> 新卒を選ぶと、大学・学部・卒業年度など、新卒特有の項目が自動で表示されます。
>
> **中途と新卒で採用フローが異なる実務に対応**した設計です。

### ユーザビリティチェック項目
- [ ] 中途/新卒の切り替えは直感的か？選択中がどちらか明確か？
- [ ] フォーム項目の数は多すぎないか？必須項目は明確か？
- [ ] 登録成功後、次に何をすべきかが案内されているか？
- [ ] 候補者一覧への戻り方は迷わないか？
- [ ] 一覧のフィルター（全体/新卒/中途）の存在に気づけるか？

## シーン 3：書類アップロード → AI解析（90秒）★核心機能①

### 操作手順
1. 候補者一覧から「田中美咲」をクリック → `/candidates/cand_001`
2. 候補者詳細ページのヘッダー「書類管理」ボタンをクリック
3. → `/candidates/cand_001/documents` に遷移
4. 「書類をアップロード」ボタンをクリック → モーダル表示
5. 書類タイプ「履歴書」を選択
6. ファイルをドラッグ＆ドロップ（デモ）
7. 「AI解析を開始」 → 6ステップのアニメーションを見せる
   - PDF読取 → テキスト構造化 → スキル抽出 → 強み分析 → Attractヒント生成 → カルテ反映
8. 解析完了 → 抽出されたスキル・経歴・資格の表示
9. 「AI統合ビュー」タブに切り替え → 全書類の統合分析結果

### ナレーション（案）
> 候補者の詳細ページから「書類管理」を開きます。
> ここで履歴書や職務経歴書をアップロードできます。
>
> アップロードすると、AIが自動で内容を解析。
> **通常30分かかる書類の読み込み作業を、AIが数秒で完了**します。
>
> ご覧ください。スキル12件、資格、職歴のハイライトが
> 構造化されたデータとして抽出されています。
>
> さらに「AI統合ビュー」では、複数の書類から抽出された情報を
> 横断的に統合。**この候補者を口説くためのヒント**まで
> AIが自動で提案してくれます。

### ユーザビリティチェック項目
- [ ] 「書類管理」ボタンの位置は見つけやすいか？
- [ ] アップロードモーダルの操作手順は明確か？
- [ ] AI解析の6ステップアニメーションは「待たされ感」がないか？
- [ ] 解析結果の情報量は多すぎないか？優先度がわかるか？
- [ ] 「AI統合ビュー」タブの存在に気づけるか？名前は直感的か？
- [ ] 「Attractアングルヒント」の意味が初見で伝わるか？
- [ ] 右サイドバーの「推奨アップロード書類」チェックリストは有用か？

## シーン 4：面接メモからシグナル抽出（90秒）★核心機能②

### 操作手順
1. 候補者詳細ページに戻る（パンくずリスト使用）
2. 「シグナル入力」ボタンをクリック → `/candidates/cand_001/signal-input`
3. 「デモデータを入力」ボタンをクリック（テキストエリアにメモが入る）
4. メモ内容を軽く見せる（面接官のフリーテキスト）
5. 「AIでシグナルを抽出する」ボタンをクリック
6. 6ステップのAI抽出アニメーション
7. 抽出結果を上から順に見せる：
   - Attractアングルのバナー
   - キャリア価値観（展開して詳細を見せる）
   - ポジティブ反応
   - 懸念事項と対応方針
   - エネルギーレベル
8. 「推奨ネクストアクション」を見せる

### ナレーション（案）
> 面接が終わったら、面接官のメモをそのまま入力します。
> **箇条書きでも、雑なメモでもOK**。
>
> AIが面接メモから「シグナル」を自動抽出します。
> シグナルとは、候補者が何を大切にしているか、
> 何に興味を示したか、何を心配しているか、という**候補者の本音**のこと。
>
> たとえば田中さんの場合、
> 「裁量の大きい環境で意思決定に関わりたい」
> 「プロダクトの成長フェーズに魅力を感じている」
> といったシグナルが検出されています。
>
> これが次のステップ——**Attract戦略の自動設計**に直結します。

### ユーザビリティチェック項目
- [ ] 「シグナル」という用語は初見ユーザーに伝わるか？
- [ ] メモ入力欄は十分な大きさか？プレースホルダーは適切か？
- [ ] 「デモデータを入力」ボタンの目的は明確か？（チュートリアル用？通常利用？）
- [ ] AI抽出後の結果画面の情報量は適切か？スクロール量は？
- [ ] 各セクション（価値観・反応・懸念）の分類は直感的か？
- [ ] 「エネルギーレベル」の意味と活用法が伝わるか？
- [ ] 「次のステップへ」の誘導は自然か？どこに行けば良いかわかるか？

## シーン 5：Attract戦略ボード（60秒）★核心機能③

### 操作手順
1. シグナル抽出結果の「Attractプランを更新」をクリック
2. → `/candidates/cand_001/attract` に遷移
3. 「メイン訴求軸（コアアングル）」の紫グラデーションエリアを見せる
4. サブ訴求軸のタグを見せる
5. 「刺さる訴求ポイント」（フィット分析）をスクロールして見せる
6. 「まだ伝えられていないこと」を見せる
7. 「ステップ別Attractアプローチ」のタイムラインを見せる（完了ステップ含む）
8. 「次回面接Attractプラン」を見せる

### ナレーション（案）
> これがATTRACTの心臓部、**Attract戦略ボード**です。
>
> 上部に「メイン訴求軸」が表示されます。
> 田中さんの場合、「プロダクト成長の意思決定者として活躍できる」が
> 最も響くメッセージだとAIが判断しています。
>
> 左側は「刺さる訴求ポイント」。候補者のシグナルと企業の魅力を
> 照合し、**マッチ度スコア**つきで表示します。
>
> 右側は「まだ伝えられていないこと」。
> 候補者に響く可能性があるが、**まだ面接で伝えていないポイント**です。
> これを意識して次の面接に臨むだけで、口説きの精度が上がります。
>
> 下部には面接ごとのステップ別アプローチ。
> **各面接で何をどう伝えるか、AIがストーリーを設計してくれます。**

### ユーザビリティチェック項目
- [ ] 「コアアングル」「サブ訴求軸」の用語は伝わるか？
- [ ] マッチスコアのバーは直感的か？数値の意味は明確か？
- [ ] 「刺さる訴求ポイント」と「まだ伝えていないこと」の違いが一目でわかるか？
- [ ] ステップ別アプローチの「完了」「次のステップ」の表示は明確か？
- [ ] 「懸念への回答方針」セクションの位置は適切か？埋もれていないか？
- [ ] 情報量が多い → 経営者が「で、結局何をすれば？」とならないか？
- [ ] 「戦略を再生成」ボタンの意味と使いどころは明確か？

## シーン 6：フィードバックレター生成（60秒）

### 操作手順
1. 候補者詳細ページに戻り「フィードバックレター」をクリック
2. → `/candidates/cand_001/feedback-letter`
3. 左側の面接リストから1回目（カジュアル面談）を選択
4. 「レターを生成」ボタンをクリック
5. 生成されたレターのプレビューを見せる
   - 合格理由セクション（ハイライト）
   - パーソナライズされたAttractメッセージ
6. 「編集」タブで内容を微調整できることを見せる
7. 「送付済みにする」ボタンを押す

### ナレーション（案）
> 面接が終わるたびに、**候補者一人ひとりに合わせた
> フィードバックレター**をAIが自動生成します。
>
> 「次の選考に進んでいただきたい理由」をシグナルに基づいて具体的に記載。
> さらに、候補者の関心に合わせた**パーソナライズされたメッセージ**が
> 自動で組み込まれます。
>
> 面接後24時間以内にこのレターを送ることで、
> 候補者に「この会社は自分をちゃんと見てくれている」という印象を与えます。
> これがAttractの本質です。

### ユーザビリティチェック項目
- [ ] 面接の選択方法（左サイドのリスト）は直感的か？
- [ ] 「レターを生成」→ 結果表示の流れはスムーズか？
- [ ] プレビューと編集の切り替えは迷わないか？
- [ ] レターの文面は自然か？（特に敬語・トーン）
- [ ] 「コピー」「PDF出力」「送付」のアクションは見つけやすいか？
- [ ] 「送付済み」ステータスの変化は視覚的に明確か？
- [ ] 右側の説明パネルは邪魔にならないか？

## シーン 7：面接官ブリーフィングシート（60秒）

### 操作手順
1. 候補者詳細 → 「面接官ブリーフ」をクリック
2. → `/candidates/cand_001/brief`
3. ブリーフヘッダー（面接日・面接官名）を見せる
4. ⚡「前回からの申し送り事項（必読）」の黄色ハイライトを見せる
5. 「今回伝えるべきキーメッセージ」をスクロール
6. 「推薦トーク例」を見せる
7. 「コピー」ボタンでテキストをコピー
8. 「面接官に送付する」ボタンを押す → 送付済み表示

### ナレーション（案）
> 次回面接の前に、**面接官ブリーフィングシート**を生成します。
>
> 最も重要なのが、この黄色い「申し送り事項」。
> 前回の面接で候補者が話した内容を、次の面接官が把握した上で臨む。
> これだけで、候補者は「この会社、ちゃんと連携されている」と感じます。
>
> さらに「今回伝えるべきメッセージ」「推薦トーク例」まで用意。
> 面接官は**準備時間ゼロ**で、戦略的な面接ができます。
>
> コピーしてSlackで共有するか、そのまま面接官に送付できます。

### ユーザビリティチェック項目
- [ ] 「申し送り事項」の黄色ハイライトは十分目立つか？
- [ ] 面接官がこのシートだけで面接に臨める情報量か？多すぎないか？
- [ ] 「推薦トーク例」の「📍シーン」表記は適切か？
- [ ] コピー → Slack共有の動線は実務的か？
- [ ] 「面接官に送付」のボタンは、何が起こるか明確か？（メール？Slack？）
- [ ] 推薦スコア（右上の数値）の表示は必要か？面接官バイアスにならないか？
- [ ] オープニングメッセージの活用シーンは伝わるか？

## シーン 8：エンディング（30秒）

### 画面
ダッシュボードに戻り、全体を俯瞰。

### ナレーション（案）
> ATTRACTは、採用担当者の勘と経験を否定するシステムではありません。
> **勘と経験を「データ」に変え、チーム全体で共有できるようにするシステム**です。
>
> 書類解析で30分、シグナル分析で20分、レター作成で15分——
> 候補者1人あたり約1時間の業務時間を削減しながら、
> **候補者が「この会社に入りたい」と思う体験**を設計します。
>
> ATTRACTで、採用を「選ぶ側」から「選ばれる側」の視点で変えていきましょう。

## 撮影・編集メモ

### 撮影環境
- **ブラウザ**: Chrome（フルスクリーン、ブックマークバー非表示）
- **画面解像度**: 1920×1080 推奨
- **ツール**: OBS Studio（無料）/ Loom（手軽） / ScreenFlow（Mac）
- **マウス操作**: ゆっくり、クリック前に一瞬止める
- **スクロール**: 急がない、セクションごとに止める

### 編集ポイント
- 各シーンの切り替え時にタイトルカード挿入
- AI解析アニメーション中はナレーションで補足（沈黙を避ける）
- 重要な画面要素にはズーム or ハイライト枠を追加
- 操作ミスは編集でカット

### 経営者向けの配慮
- 専門用語は初出時に必ず説明
- 「つまり○○ということです」で要約を挟む
- 数字（時間削減、スコア）を強調
- 「従来の方法だと〜」→「ATTRACTなら〜」の対比構造

## 改善アイデア（動画作成後に検討）

以下は動画撮影中に感じた違和感をメモし、撮影後に検討する項目：

1. **オンボーディングウィザード**: 初回ログイン時のガイドツアー
2. **ステータスバー**: 候補者ごとの進捗（「あと○ステップ」）
3. **通知**: 「フィードバックレター未送付（24h経過）」のアラート
4. **用語のツールチップ**: 「シグナル」「Attract」にマウスオーバーで説明表示
5. **プリント対応**: ブリーフシートのPDF出力レイアウト最適化
